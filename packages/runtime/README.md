# @ggpwnkthx/dom-runtime

Client DOM mount and patch behavior.

## Status

**Reserved** — no runtime logic implemented yet. This package exists to establish package boundaries.

## Purpose

- Client-side DOM element creation and mounting
- Virtual DOM diffing and patching
- Event binding and delegation
- Component lifecycle hooks

## Note

This package should remain focused on client DOM behavior. SSR hydration belongs in `@ggpwnkthx/dom-hydrate`.
